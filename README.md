# AI Web Scout

Сервис пакетного анализа сайтов: на вход список доменов + промпт «что найти»,
на выходе — извлечённые данные по каждому домену с указанием источника.

## Интерфейс (Clay-style таблица)

Новый интерфейс — табличный, в духе Clay: **домены в левом столбце,
AI-промпт-столбцы справа**, результат по каждому домену в ячейке. Можно
добавлять несколько столбцов, у каждого — свой промпт на весь столбец.

```
pip install -r requirements.txt
export OPENAI_API_KEY=sk-...        # либо ввести в UI → Настройки
python server.py                   # → http://127.0.0.1:8000
```

- `server.py` — тонкий Flask-бэкенд.
- `store.py` — хранение состояния в SQLite (`scout.db`); переживает рестарт.
- `web/index.html` — фронтенд (self-contained, без сборки).
- `pipeline.py` — обёртка над движком: `keywords → scrapy → LLM-воркеры`.

Домены, столбцы и результаты сохраняются в `scout.db` (SQLite) и доступны после
перезапуска сервера. Путь к БД переопределяется переменной `SCOUT_DB`. API-ключ
на диск **не** пишется — живёт в памяти процесса / `OPENAI_API_KEY`.

### Как пользоваться

1. **Импорт доменов** — вставьте список (по одному на строку).
2. **+ Столбец** — задайте промпт (можно с переменными `{{Company Name}}`,
   `{{Website}}`), выберите модель.
3. **Запустить** на столбце (или **Запустить всё**) — пройдут 3 стадии:
   генерация ключевых слов → краулинг → LLM-извлечение.
4. Клик по ячейке — детали: значение, `source_url`, промпт, как искалось.
5. **Экспорт** — CSV (Google Sheets — через `sheets_export.py`).

## Движок (не менялся)

Логика поиска данных та же, что и раньше, переиспользуется как есть:

| Стадия | Файл |
|---|---|
| Генерация ключевых слов | `llm_validator.py → get_keywords_from_query()` |
| Краулинг (Scrapy, 5 стр./домен, приоритет contact/about/team) | `scraper/spiders/site_spider.py` |
| LLM-извлечение `{found, data}` (GPT-4o-mini) | `llm_validator.py → validate_with_llm()` |
| Параллельные воркеры (по доменам) | `run_llm_workers.py` |

Каждый запуск столбца = один прогон движка под один промпт. Запуски
синхронные, один за раз (без очереди задач и кэша — по согласованному объёму).
Состояние персистится в SQLite (`store.py`).
