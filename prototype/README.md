# AI Web Scout — прототипы интерфейса (Clay-стиль)

В папке два self-contained прототипа (фронтенд-макеты с моковыми данными,
**только дизайн**, без бэкенда):

| Файл | Что это |
|---|---|
| `scout-clay-ui.html` | Базовый скаут: домены — строки, AI-промпт-столбцы — справа. |
| `scout-unified-ui.html` | **Объединённый сервис**: поиск компаний (GrowthLoop) + скаут в одном интерфейсе. |

---

## 🔗 Объединённый прототип — `scout-unified-ui.html`

Главная идея, как в Clay: **поиск компаний и AI-обогащение живут в одном
экране**. Сервис поиска компаний (репозиторий `growthloop-app`) встроен слева
как **источник строк**, его результаты сразу попадают в ту же Clay-таблицу,
где работает скаут.

```
prototype/scout-unified-ui.html
```

### Поток работы (один экран)

1. **Найти компании** (левая панель «Источник») — три метода из growthloop-app:
   - **Фильтры** (реестр ФНС): ОКВЭД, регион, выручка от/до, сотрудники,
     с телефоном/email, только действующие.
   - **Вакансии HH.ru**: ключевое слово, город, только в названии, даты,
     макс. страниц.
   - **Список ИНН** (OFDATA): вставка ИНН с живыми метриками (фрагменты /
     кандидаты / уникальные / дубликаты), проверка контрольных сумм.
2. **Превью результатов** — модалка с чекбоксами и «Выбрать все»,
   «Добавить N в таблицу» (как `st.data_editor` в growthloop-app).
3. Компании становятся **строками** таблицы с нативными полями из поиска
   (Источник · Регион · Отрасль/ОКВЭД · Выручка · Сотрудники · Телефон ·
   Email · ИНН).
4. **Скаут** запускается поверх найденных компаний: AI-столбцы и тот же
   3-стадийный пайплайн (`Ключевые слова → Краулинг → LLM`).

### Как читается «слияние» в UI

- Два **бэнда-заголовка** над таблицей: «Из поиска компаний» (indigo) и
  «Скаут · AI-обогащение» (teal) — видно, какие столбцы откуда.
- **Бейдж источника** в каждой строке: Реестр / HH.ru / ИНН / Импорт.
- Клик по строке → **карточка компании**: источник, все данные поиска и
  результаты скаута вместе + «Запустить скаут по этой компании».
- **Настройки** объединяют ключи обоих сервисов: `FNS_API_KEY`,
  `OFDATA_API_KEY`, `HH_API_KEY` (поиск) и `OpenAI API Key` + лимиты (скаут).
- **Экспорт CSV** включает и поля поиска, и AI-столбцы с `source_url`.

## Базовый прототип — `scout-clay-ui.html`

Табличный формат, как в Clay: **домены — в левом столбце, AI-промпт-столбцы —
справа**, результат по каждому домену в ячейке. Это **только дизайн интерфейса**
(фронтенд-макет с моковыми данными), без подключения к реальному бэкенду.

## Как посмотреть

Открыть файл в браузере:

```
prototype/scout-clay-ui.html
```

Просто двойной клик по файлу или:

```bash
# из корня репозитория
python3 -m http.server 8000
# затем открыть http://localhost:8000/prototype/scout-clay-ui.html
```

Один self-contained HTML-файл, без сборки и зависимостей.

## Что можно потрогать

- **Запустить столбец** — кнопка «Запустить» в шапке столбца. Показывает все
  3 стадии текущего пайплайна: `Ключевые слова → Краулинг → LLM-извлечение`,
  ячейки заполняются по очереди (found / ❌ не найдено + source_url).
- **Запустить всё** — прогоняет все столбцы сразу.
- **Добавить столбец** (`+ Столбец`) — выбор типа (AI-промпт / скрейпинг /
  обогащение / формула), промпт на весь столбец с переменными
  `{{Company Name}}` и `{{Website}}`, выбор модели, готовые шаблоны.
- **Редактировать промпт** — клик по строке промпта в шапке столбца.
- **Клик по ячейке** — боковая панель с результатом, source_url, промптом,
  моделью и «следом краулинга» (какие страницы обошёл робот).
- **Импорт доменов** — вставка списка (по одному на строку), как сейчас.
- **Экспорт** — CSV (реально скачивается) и макет выгрузки в Google Sheets.
- **Меню столбца** (⋯) — запуск / редактирование / дублирование / удаление.
- **Поиск** по доменам, тёмная тема (иконка солнца).

## Что отражено из текущего сервиса

| Текущая возможность | Где в прототипе |
|---|---|
| Загрузка списка доменов | Левый столбец + «Импорт доменов» |
| Запрос «что ищем» (free-form) | Промпт на весь столбец |
| Шаблоны `{{Company Name}}`, `{{Website}}` | Чипы-переменные в редакторе промпта |
| Stage 1 — генерация ключевых слов | Полоса прогресса пайплайна + чипы ключевых слов |
| Stage 2 — краулинг (5 стр./домен) | Стадия 2 + «след краулинга» в панели ячейки |
| Stage 3 — LLM-извлечение (GPT-4o-mini) | Стадия 3, бейдж модели на столбце |
| Результат: found / ❌ Не найдено + source_url | Содержимое ячеек и боковая панель |
| OpenAI API Key, воркеры, лимиты страниц | Модалка «Настройки» |
| Экспорт CSV / Google Sheets | Модалка «Экспорт» |

## Дизайн-система (из скилла ui-ux-pro-max)

- Палитра: teal `#0D9488` (основной) + orange `#EA580C` (акцент/CTA)
- Стиль: micro-interactions, плотная data-table вёрстка
- Светлая и тёмная темы, доступные контрасты, hover/focus-состояния
